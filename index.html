Nobara Gaming Guide

Quick‑start Steam & Proton‑GE Battle.net RuneLite Minecraft Streaming Fixes

Why Nobara?

Nobara is a Fedora‑based distro tuned for gaming and media. It removes a ton of setup friction: recent kernels, multimedia bits, and sane defaults. This guide shows a smooth path—from USB creation and install, to Steam/Proton‑GE, Battle.net (WoW), RuneLite, Discord/Spotify, Minecraft, Sunshine streaming, HiDPI scaling, and a simple backup/restore approach.

Quick‑start (ISO -> USB -> Install)

  1. Download Nobara: Nobara download page
  2. Create a bootable USB (recommended: Ventoy): Ventoy lets you install to a USB once, then you just copy ISO files onto it like a normal drive. Boot menu shows your ISOs automatically.

    On Windows

    1. Download Ventoy (Windows EXE), run as Admin.
    2. Select your USB → Install.
    3. Copy the Nobara .iso onto the Ventoy USB (drag/drop).

    On Linux

    1. Extract Ventoy release; run the installer script against your USB device.
    2. Mount the USB; copy the Nobara .iso onto it.
  3. Boot the USB on an ASUS ROG: Power off. Hold ESC and tap Power to open the boot menu; pick the USB. (F2 enters BIOS.)
  4. Install Nobara, then reboot. Leave the USB inserted until the installer tells you to remove it.

Tip: If the USB doesn’t show up, in BIOS disable Fast Boot and check Secure Boot/CSM settings.

Steam + Proton‑GE (for stubborn Windows games) ▶

Install Steam from Nobara’s software center (or sudo dnf install steam). For better Windows game compatibility, install Proton‑GE using ProtonUp‑Qt, then force that version per game in Steam.

Install ProtonUp‑Qt

# Flatpak (system scope)
flatpak install --system flathub net.davidotek.pupgui2

# Run it and install the latest Proton-GE
flatpak run net.davidotek.pupgui2

In Steam → Game Properties → Compatibility → “Force the use of a specific Steam Play compatibility tool” → pick Proton‑GE.

NVIDIA + Flatpak driver runtime

For Flatpak games to use NVENC and your GPU, install the NVIDIA GL runtimes that match your host driver: 

ver=$(nvidia-smi --query-gpu=driver_version --format=csv,noheader | tr . -)
flatpak install --system flathub org.freedesktop.Platform.GL.nvidia-$ver
flatpak install --system flathub org.freedesktop.Platform.GL32.nvidia-$ver
flatpak update
Battle.net (WoW) via Steam + Proton‑GE ▶
  1. Download the Battle.net installer EXE.
  2. In Steam: Games → Add a Non‑Steam Game… → Browse and select the EXE (or add a dummy EXE then edit).
  3. Right‑click it → Properties → Compatibility → force Proton‑GE.
  4. Launch once; complete Battle.net install; then install WoW inside Battle.net.

If the “agent sleeps” or the installer briefly opens then closes, double‑check you’re forcing Proton‑GE, and try a newer/older Proton‑GE build in ProtonUp‑Qt.

RuneScape / RuneLite (Flatpak) ▶

Install RuneLite

flatpak install --system flathub net.runelite.RuneLite

HiDPI scaling: increase Java UI scale (e.g., 150%): 

flatpak override --user --env=_JAVA_OPTIONS='-Dsun.java2d.uiScale=1.5' net.runelite.RuneLite

Jagex Launcher (optional)

There’s a community Flatpak helper for the Jagex Launcher. It works but is unofficial—use at your own discretion. 

curl -fSsL https://raw.githubusercontent.com/USA-RedDragon/jagex-launcher-linux-flatpak/main/install.sh | bash
flatpak install --user JagexLauncher com.jagex.Launcher.ThirdParty.RuneLite

If tray icons stick or black squares appear, restarting Plasma’s tray helps: pkill xembedsniproxy; kquitapp6 plasmashell; kstart6 plasmashell

Discord, Spotify & audio (Scarlett Solo) ▶

Install

flatpak install --system flathub com.discordapp.Discord
flatpak install --system flathub com.spotify.Client

If the mic doesn’t appear in Discord, try Settings → Voice & Video → Reset Voice Settings, re‑select your input, or restart Discord. Use pavucontrol to pick the Scarlett Solo as the default input.

HiDPI & KDE

Set system scale in System Settings → Display (e.g., 150%). Flatpaks respect the desktop scale. For occasional apps that don’t, use app‑specific overrides (like the RuneLite example).

Minecraft (Java + Bedrock options) ▶

Java Edition (recommended)

Use Prism Launcher for a clean, stable modded experience:

flatpak install --system flathub org.prismlauncher.PrismLauncher

Prism lets you manage multiple instances, versions, and modpacks easily.

Bedrock (Linux choices)

  • Waydroid (Android container): install the Play Store version you own; good compatibility.
  • mcpelauncher (unofficial): works for some versions; read notes and requirements carefully.

Cross‑play with Xbox: Bedrock clients (Android/Waydroid) can join the same Bedrock servers/Realms as Xbox if the versions match. Ensure NAT/ports are open on your server if hosting.

Stream to TV (Sunshine → Moonlight, incl. Apple TV) ▶

Sunshine runs on your Nobara PC; Moonlight connects from TVs/phones/tablets (including Apple TV). Great latency and quality (NVENC/AV1 on RTX).

Install Sunshine (Nobara/Fedora)

# LizardByte stable COPR (RPM)
sudo dnf copr enable -y lizardbyte/stable
sudo dnf install -y Sunshine

# Allow KMS capture (Wayland): grant CAP_SYS_ADMIN to the binary
sudo setcap cap_sys_admin+p "$(readlink -f "$(which sunshine)")"

# Start (user service, optional)
systemctl --user enable --now sunshine

If you use the Flatpak build, also install the matching Flatpak NVIDIA GL runtimes (see Steam section).

Client & pairing

  • Install Moonlight on Apple TV / iOS / Android.
  • Pair while on the same LAN first; then test over Internet if desired.
  • In Sunshine’s web UI, add “Desktop” and your favorite games.
Back up & restore saves (simple CLI) ▶

Lightweight folder backup (compatdata + userdata)

# Set game name
GAME_NAME="REPO"   # or your title as shown in Steam

# Resolve Steam dir
STEAM_DIR="$HOME/.local/share/Steam"; [[ -d "$HOME/.steam/steam/steamapps" ]] && STEAM_DIR="$HOME/.steam/steam"

# Find appid by name
APPID=$(grep -RiEl "\"name\"[[:space:]]*\"$GAME_NAME\"" "$STEAM_DIR/steamapps"/appmanifest_*.acf | sed -E 's/.*appmanifest_([0-9]+)\.acf/\1/' | head -n1)

# Find SteamID that has this app's userdata
STEAMID=$(basename "$(find "$STEAM_DIR/userdata" -mindepth 2 -maxdepth 2 -type d -name "$APPID" -print -quit | xargs dirname)")

STAMP=$(date +%Y%m%d-%H%M)
DEST="$HOME/Backups/${GAME_NAME}-${STAMP}"
mkdir -p "$DEST/compatdata/$APPID" "$DEST/userdata/$APPID"

nice -n 19 ionice -c3 rsync -aH --info=progress2 "$STEAM_DIR/steamapps/compatdata/$APPID/" "$DEST/compatdata/$APPID/"
nice -n 19 ionice -c3 rsync -aH --info=progress2 "$STEAM_DIR/userdata/$STEAMID/$APPID/" "$DEST/userdata/$APPID/"
echo "Backup folder: $DEST"

Restore (example for REPO saves inside the Proton prefix)

# Choose the backup you want to restore
BACKUP_NAME="REPO-20251011-0141"

STEAM_DIR="$HOME/.local/share/Steam"; [[ -d "$HOME/.steam/steam/steamapps" ]] && STEAM_DIR="$HOME/.steam/steam"
APPID_REAL=3241660   # REPO

BACKUP="$HOME/Backups/$BACKUP_NAME"
SRC=$(find "$BACKUP/compatdata" -type d -path '*/pfx/drive_c/users/*/AppData/LocalLow/semiwork/Repo' | head -n1)  # Repo or Repo/Saves, both fine

# Stop Steam so it doesn't race the restore
steam --shutdown >/dev/null 2>&1 || true; for i in {1..20}; do pgrep -x steam >/dev/null || break; sleep 0.2; done

# Live target
TARGET="$STEAM_DIR/steamapps/compatdata/$APPID_REAL/pfx/drive_c/users/steamuser/AppData/LocalLow/semiwork/Repo"
TS=$(date +%Y%m%d-%H%M)
[ -d "$TARGET" ] && mv "$TARGET" "${TARGET}.bak-$TS" || true
mkdir -p "$TARGET"
rsync -aH --info=stats2 "$SRC"/ "$TARGET"/
echo "Done. Launch the game (Steam Cloud off if applicable)."

Steam Cloud can overwrite local changes. Disable it when testing a rollback.

Troubleshooting snippets ▶
  • ASUS boot menu: hold ESC while powering on. BIOS: F2.
  • Battle.net installer closes immediately: force Proton‑GE; try a different GE build in ProtonUp‑Qt.
  • Flatpak games have no GPU encode (NVENC): install matching org.freedesktop.Platform.GL(.32).nvidia-<driver> runtimes.
  • Discord mic missing: Reset Voice Settings, re‑select input, check pavucontrol.
  • Plasma tray icons “stick” or show black squares: pkill xembedsniproxy; kquitapp6 plasmashell; kstart6 plasmashell.
  • Sunshine on Wayland can’t capture: ensure sudo setcap cap_sys_admin+p "$(readlink -f "$(which sunshine)")", and matching Flatpak NVIDIA runtimes if using the Flatpak build.
